# Ritual: Prioritization (per cycle, 60 min)

Criteria #5: consistent and transparent — the ranking matters less than the published rationale.

## Criteria (score each request 1-3 on each; publish the sheet)

1. **Criterion fit** — maps to one of the 8 (which?)
2. **Demand evidence** — discovery quotes / repeated asks / usage data (speculation = 1)
3. **Cognitive-load delta** — how much thinking does it remove for how many Developers?
4. **Operating cost** — managed capability (3) … self-operated component (1) [the kit thesis]

## Output

- Ranked list WITH scores visible to all Dev Teams (publish in #platform-help or the repo)
- Every "no" or "later" gets its reasoning written — a transparent no builds more trust than
  a silent yes that never ships
- Deferred items retain their evidence so re-proposal is cheap
